HTML-常用HTML标签

发表于 2019-01-17 | 更新于 2019-03-14

今天来简单了解一下些常用的HTML标签：

  • a
  • form
  • input
  • button
  • h1
  • p
  • ul
  • ol
  • small
  • strong
  • div
  • span

<a>元素

<a>可以创建一个到其他网页、文件、同一页面内的位置、电子邮件地址或任何其他URL的超链接。


download

此属性指示浏览器下载URL而不是导航到它，因此将提示用户将其保存为本地文件。




herf

包含超链接指向的URL或URL片段。其中URL片段是指用’#’标记的当前页面中某个HTML元素的ID。可以使用’#top’或者’#’来返回页面顶部。

target

该属性指定在何处显示链接的资源。 取值为标签（tab），窗口（window），或框架（iframe）等浏览上下文的名称或其他关键词。

  • _self：在当前页面打开。如果没有指定target属性，此值是默认属性。
  • _blank：新窗口打开。
  • _parent：加载响应到当前框架的HTML4父框架或当前的HTML5浏览上下文的父浏览上下文。如果没有parent框架或者浏览上下文，此选项的行为方式相同_self。
  • _top：加载的响应成完整的，原来的窗口，取消所有其它frame。 HTML5中：加载响应进入顶层浏览上下文（即，浏览上下文，它是当前的一个的祖先，并且没有parent）。如果没有parent框架或者浏览上下文，此选项的行为方式相同_self。



<form>元素

<form>元素表示了文档中的一个区域，这个区域包含有交互控制元件，用来向web服务器提交信息。


action

一个处理这个form信息的程序所在的URL。这个值可以被 <button> 或者 <input> 元素中的 [formaction](https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/button#attr-formaction) 属性重载（覆盖）。

method

设置用哪种HTTP方式来提交form，可能的值有：

  • GET：指的是 HTTP POST 方法 ; 表单数据会包含在表单体内然后发送给服务器.
  • POST：指的是 HTTP GET 方法; 表单数据会附加在 action 属性的URI中，并以 ‘?’ 作为分隔符, 然后这样得到的 URI 再发送给服务器. 当这样做（数据暴露在URI里面）没什么副作用，或者表单仅包含ASCII字符时，再使用这种方法吧。

如果未指定该属性，默认是GET方法。

<input>元素

所有<input>元素需要被表单提交都需要指定name属性，并且如checkout和radio这两个类型的<input>元素需要将name属性设置成一样来表示同一组数据。其他具体各个<input>元素的用法大家可以自行去MDN查看。
主要这里是引出另一个<label>元素：它通常关联一个控件，或者是将控件放置在label元素内，或者是用作其属性。这样的控制称作label元素的labeled control 。例如，当天包裹住文本输入的<input>元素时，点击它会让输入框获得焦点；当它包裹类型为checkbox和radio的<input>元素时，点击它如同点击checkbox和radio,会触发选中或者取消选中事件。
